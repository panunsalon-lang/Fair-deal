## 3. Database Initialization

If you haven't initialized your database yet, the login will fail because the tables don't exist. You can fix this by running these commands in your local terminal:

```bash
# 1. Install dependencies
pnpm install

# 2. Push the schema to your database
# (Ensure your local .env has the correct DATABASE_URL)
pnpm db:push

# 3. Create/Reset your admin credentials
node update-admin-creds.mjs
```

## 5. Troubleshooting the JSON Error

If you still see the error:
1. **Check Vercel Logs**: Go to your Vercel Dashboard > Deployments > [Current Deployment] > Logs. Look for any error messages starting with `[Auth]` or `[Database]`.
2. **Database Connectivity**: Ensure your database is accessible from the internet (TiDB allows this by default).
3. **Redeploy**: After updating environment variables, you **must** redeploy your project on Vercel for the changes to take effect.
